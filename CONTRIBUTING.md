## Pull Request Process

1. **Fork** the repository
2. **Create a branch** (`git checkout -b improve-linear-algebra-section`)
3. **Make your changes** with clear commit messages
4. **Test all links** you add or modify
5. **Submit PR** with description of changes
6. **Respond to feedback** from maintainers

## Style Guidelines

### Writing Style
- **Clear and concise** - avoid jargon
- **Encouraging** - maintain positive, motivating tone
- **Realistic** - honest about difficulty and time
- **Inclusive** - write for diverse backgrounds

### Formatting
- Use **Markdown** formatting consistently
- Keep **line length** reasonable (~80-100 chars)
- Use **emoji sparingly** (only for major section headers)
- Follow existing **heading hierarchy**

### Link Format
- Use descriptive link text: `[MIT 18.06 Linear Algebra](url)`
- NOT: `[click here](url)`
- Verify links work before submitting

## Time Estimate Guidelines

If suggesting time estimate changes:
- Base on **actual experience** completing the material
- Include **problem sets and projects**
- Account for **different pace levels**
- Provide range (min-max hours)
